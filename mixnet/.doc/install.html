Installing Vfork

Below we detail three simple steps for installing a pure Java build of Vfork on 64-bit Ubuntu 10.04 (Lucid) with a Bash-shell. These instructions should be easy to adapt to other Un*xes and shells. You may alternatively install Vfork with support for optimized native code and/or other extensions.

1. Download, Build, and Install Vfork.

If you are running a vanilla installation of some Un*x, then try the following commands, which fetches, compiles, and installs a pure Java version of Vfork. 

  wget http://www.mixnet.com/download/mixnet-VFORK_VERSION.tar.gz
  tar xvf mixnet-VFORK_VERSION.tar.gz
  cd mixnet-VFORK_VERSION
  ./configure
  make
  sudo make install

2. Set Up Environment.

If you are running a vanilla installation of some Un*x with Bash as login-shell, then you can simply copy the file .mixnet_env (generated by make) to your home directory and source this. You can achieve this as follows. 

  cp .mixnet_env ~/
  printf '\nsource ~/.mixnet_env' >> ~/.bashrc
  source ~/.mixnet_env

3. Initialize the Random Source.

Some commands need randomness, and you need to tell Vfork which source of randomness to use. You may use /dev/urandom (the example below), one of mixnet's pseudo-random generators, or some other source of randomness provided as a readable device. More information can be found in the README file. You can use the following to securely initialize your random source in most Un*xes. 

  vog -rndinit RandomDevice /dev/urandom

Uninstall.

Assuming you followed the above installation instructions and want to uninstall, you need to remove the .mixnet_env from your home directory, remove the line sourcing .mixnet_env from your .bashrc and then simply execute: 

  sudo make uninstall
