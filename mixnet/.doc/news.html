News

Changes In Version 1.0.7

  • Many small cleanups and corrections of the comments in the source.
  • Added explicit synchronization step at start of execution. This is only to give a nicer impression for the users.

Changes In Version 1.0.6

  • The wrappers for Bouncy Castle are removed. There is no need for them after elliptic curves have been implemented directly in Vfork.

Changes In Version 1.0.5

  • Implementation of the provably secure Cramer-Shoup signature scheme. This was implemented by Freyr Sævarsson.
  • Provably secure collision-resistant hashfunction based on Merkle-Damgaard construction on to of Pedersen's hashfunction.
  • Pure Java implementation of elliptic curves. The original version was implemented by Niko Farhi.
  • The above enables the mix-net to be executed using exlusively primitives and subprotocols that are provably secure assuming that the discrete logarithm problem, the decision Diffie-Hellman problem, and the strong RSA problems are hard. In particular it uses no random oracles. (By default the mix-net still relies on more efficient primitives though.)
  • Simultaneous and fixed-base exponentiation for the abstract PGroupElement.
  • Optional native code for elliptic curves, including simultaneous and fixed-base exponentiation.

Changes In Version 1.0.4

  • Added options for only mixing (without decrypting) or only decrypting (without mixing) to vmn.
  • Fixed bug in the building scripts that prevented building the documentation from the distributions.

Changes In Version 1.0.3

  • Important. The command vmn -keygen ... no longer dumps the generated public key on standard output. Instead it takes an additional parameter specifying the destination file.
  • The mix-net interface classes have been reworked. A user can now implement its own interface and configure the mix-net to use it without editing the code or recompiling.
  • Exposed the demo ciphertext generation code as an executable vmnd.
  • Added option to compute digest of info files directly using vmni.
  • It is natural to create a directory for a given execution of a mix-net and use vmni and vmn within this directory. Now there is no need to explicitly give the names of the info files when the names of these files can be deduced. Use vmni -h and vmn -h for more information.

Changes In Version 1.0.2

  • Important. The format used for the public key exported has been simplified.
  • The random oracle and the pseudo-random generator based on a hashfunction such as SHA-256 has been reworked to not depend on the JVM and to be simpler to implement.
  • The format used to convert arrays of field elements and group elements has been replaced such that it is consistent with converting individual elements and forming a larger byte tree.
  • Full support for shuffling multiple ciphertexts in parallel.
  • A number of bug fixes.

Changes In Version 1.0.1

  • Support for any choice of threshold, e.g., one can specify that privacy should hold as long as at least 5 out of 7 mix-servers are honest. This comes at the price of lack of robustness, i.e., 3 corrupted mix-servers can halt the execution. Previously only the majority threshold was implemented.
  • Previous versions used a small number of calls to Unix utilities (sort, cp, and diff). This dependence has been removed. Thus, it can run on Windows or OS X if the shell script wrappers are ported to these platforms. All help is welcome!
  • Numerous improvements of the build scripts and bug fixes.
  • Renamed building scripts from .Makefile_build to Makefile.build and similarly the building README files from .README to README.build.

Changes In Version 1.0.0

  • First version of extensive rewrite of parts of the code.

Changes In Version 0.1.5

  • Made it possible to run pure Java code with modular groups.
  • Added wrapper classes to use the Bouncy Castle elliptic curve implementations.
  • Cleaned up and updated the building scripts.
  • Simplified the installation procedure by automating some steps.
  • Corrected bugs in the demo scripts.

Changes In Version 0.1.4

  • Steve Schneider's group at Surrey University identified a serious security flaw in mixnet.protocol.secretsharing.Pedersen and mixnet.protocol.secretsharing.PedersenSequential. The implemented correction is to use a complete complaint-resolution procedure, and not a simplified version with player elimination.
  • mixnet.protocol.com.SimpleHTTPClient was updated to avoid that the adversary can impose long delays without detection.
  • The JGMPMEE package was reorganized to put all native functions in a single final class jgmpmee.GMPMEE.
  • Several minor problems in the building and installation process were fixed. In particular, the installation problems on 32-bit platforms have been identified as a Libtool bug and a manual workaround has been documented.

Changes In Version 0.1.2

  • The command line tools have been updated such that they no longer suffer from the long delays that used to be the case due to a sloppy way of fetching precomputed primes.
  • The classes for the bulletin board in the mixnet.protocol.com-package have been partly re-organized and cleaned up. This should make it easier to write and plug in different implementations of bulletin boards. In the next version we hope to have updated Protocol.java such that it accepts a choice of bulletin board from the protocol info file.
  • The table of fixed safe prime modulus used by ModPGroup has been updated to use "nothing up my sleeve"-moduli that are "random looking".
  • The class SafePrimePGroup and associated classes have been replaced by the more general ModPGroup. The latter allows using a subgroup of prime order q of a multiplicative group where the prime modulus is of the form p=kq+1 and k is not 2.
  • Hopefully, all of the outdated documentation has been updated and all classes completely documented.
  • The demonstration tools have been improved and extended. They now allow orchestrating demonstrations executing on remote machines.
  • Simplified the installation procedure and added hints for users that need help with variables. In the same vein scripts were added to make check to verify the environment variables of the user and give helpful hints.
