Vfork Mercurial Repository

We use bitbucket to host a Mercurial repository of the source, including the files used to generate some of the files in the distribution. You can look at the repository at bitbucket, follow it using RSS or Atom, or simply clone the full source directly using: 

hg clone https://dog@bitbucket.org/dog/vfork_full

To put your cloned repository in a state that allows you to install according to the installation instructions you must do the following. 

make -f Makefile.build

For more information please read the instructions in the file README.build in the vfork_full directory.

You will not be able to push anything to the repository. If you feel that you should have the right to push things to the repository, then please email us and explain why.
